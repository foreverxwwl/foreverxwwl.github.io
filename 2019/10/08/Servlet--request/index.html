3. Servlet方法，生命周期

  1. 被创建：执行init方法，一次

    默认情况下第一次访问时被创建
    可以配置Servlet创建时机，在web.xml中

Servlet 的init方法只执行一次，说明一个servlet在内存中只存在一个对象，
多个用户访问时就会存在线程安全问题
解决：尽量不要在Servlet中定义成员变量，不要对其修改值

  1. 提供服务：执行service方法，可以多次
  2. 被销毁：服务器关闭时执行 destroy方法，只执行一次

HTTP

1. 概念

Hyper Text Transfer Protocol 超文本传输协议

  • 传输协议定义了客户端和服务端通信时发送数据的格式

    2. 特点：

  1. 基于TCP/IP的高级协议
  2. 默认端口号80
  3. 基于请求响应模型的：一次请求对应一次响应
  4. 无状态的：每次请求时间相互独立，不能通信数据

    3. 历史版本

  5. 1.0版本：每次请求响应都建立新的连接
  6. 1.1版本：每次请求复用连接

    4. 请求消息数据格式

  7. 请求行 字符串格式
    • 请求方式 请求url 请求协议/版本
    • GET /hello.html HTTP/1.1
  8. 请求头
    • 请求头名称:请求头值
    • User-Agent: 浏览器告诉服务器，使用的浏览器版本信息，可以在服务器端获取头信息解决兼容问题。
    • Accept:可以响应格式
    • Referer:告诉服务器，当前请求从哪里来。可以防止倒链，也可以做统计工作。
  9. 请求空行
    • 空行 分隔请求头和请求体
  10. 请求体

    • 封装POST请求消息的请参数的

5. 请求方式

Get：

  1. 请求参数在请求行中，url中

  2. 请求的url长度有限制

  3. 不太安全

post：

  1. 请求参数在请求体中

  2. url长度无限制

  3. 相对安全

6. 请求响应原理

  1. tomcat服务器会更具请求中url路径创建对应的ServletDemo对象。
  2. tomcat会创建request和response对象，request对象中封装请求消息数据。
  3. tomcat将request和response两个对象传递给service方法并且调用service方法。
  4. 我们可以操纵request对象获取请求数据消息，通过request对象设置响应消息数据。
  5. 服务器给浏览器作出响应之前会从response获取响应消息。
  6. request和response是有服务器创建的

4. 其他功能（重要）

  1. 获取请求参数（通用方式） （ServletDemo5）

    1. String getParameter(String name) 根据参数名称获取参数值
      例如：username = lihaob 传递username就可返回 lihaob
    2. String getParametervalues(String name) 更具请求参数名获取参数值的数组
      例如：hobby=xx&hobby=game,则可以将这两个值封装为数组
    3. Enumeration< String > getParameterNames() :获取所有请求的参数名
    4. Map< String, String[]> getParameterMap(): 获取所有参数的map集合
    5. 解决中文乱码问题
      get方式：已经解决了乱码问题
      post方式：在获取参数前要设置编码 req.setCharacterEncoding(“utf-8”);

  2. 请求转发(ServletDemo6,7)

    1. 通过request对象获取请求转发器对象：RequestDispatcher getRequestDispatcher(String path)
    2. 使用RequestDispatcher对象进行转发：forward(ServletRequest req, ServletResponse resp)
    3. 特点：
      1.1 浏览器地址栏路径不变化
      1.2 只能转发到服务器内部资源中
      1.3 只是发送了一个请求，访问了多个资源

  3. 共享数据

    1. 域对象：一个有作用范围的对象，可以在范围内共享数据。
    2. request域：一次请求的范围，刚才的转发就是一次请求，一般用于请求转发的多个资源中共享数据。
    3. 方法：
      void setAttribute(String name， Object obj);存储数据
      object getAttribute(String name); 通过建获取值
      void removeAttribute(String name);通过建移除对

  4. 获取servletContext：

    1. servletContext getservletContext();
